## What happened (2026 compression wave)

1. Dozens of **`GENE_*_GEN2.md`** files shared one story with newer **umbrella `*.gen1.md`** files.  
2. Stubs kept **stable genetic tags** for `genes_config.json`, resolver, and old links.  
3. Long prose moved to: **umbrella body**, **cluster synergy tables**, **journals**, or **git history**.

Compression optimizes **navigation**; it must not erase **judgment**. Judgment lives in [foundation.core_pillars.gen1.md](../genes/foundation.core_pillars.gen1.md) and linked long-forms.

## Stub ≠ “deprecated philosophy”

A stub that only says “see umbrella X” is **routing**, not deletion. Tools resolve to umbrella at load time ([gene_document_resolver.py](https://github.com/agentstacktech/AgentStack/tree/main/philosophy/gene_document_resolver.py)).

**Do not** resurrect duplicate full copies of umbrella text into stubs — update the **`.gen1`** canonical file instead ([repo.engineering.controlled_changes.gen1.md](../genes/repo.engineering.controlled_changes.gen1.md)).

## Maintenance

When a pillar’s meaning shifts (new ADR, new contour):

1. Update the **foundation.*.gen1** file first.  
2. Add one row to PHILOSOPHY_INDEX **only** if founders need a new “insight” block.  
3. Touch heritage stubs **only** to fix broken redirects.
